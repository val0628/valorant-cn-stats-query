# Changelog

## 2026-07-22

- 公共近期战绩、房间分析和当前对局页面统一支持行内展开单局详情，不再需要离开当前页面查看双方 10 人数据。
- 单局详情展示双方阵营、英雄、段位、K/D/A、ACS/分数和结果；上游详情暂时不可用时，会根据已有比赛数据补全可展示内容，提高详情覆盖率。
- 玩家摘要新增当前赛季爆头率，以及根据头、身体、腿部实际命中次数计算的赛季命中习惯。
- 单局 10 人详情新增单局爆头率和头/身/腿命中分布。爆头率按头部命中数占全部命中数计算，不使用爆头击杀数推测。
- 英雄和地图名称统一为中文，详情补充玩家昵称，减少“未知玩家”和中英文混排。
- 疑似组队分析改为默认折叠，只有用户主动点击后才显示名单、彩色关联标记、共同场次、友方/敌方关系和共同对局样例。
- 公共实时页面进一步隐藏内部玩家标识与比赛标识，只保留玩家可理解的战绩和分析信息。
- 更新 GitHub 首页、使用教程、FAQ、Roadmap 与真实脱敏截图，展示新版当前对局、单局命中分布和疑似组队分析。

## 2026-07-21

- 新增独立使用审计数据库，记录公共查询的访问 IP、查询内容、关联账号及处理结果。
- 实时账号选择、房间近 3 场和当前对局查询新增账号使用记录，包括访问 IP、游戏会话来源 IP、用户选择的账号、入口和结果。
- 使用记录默认保留 180 天并自动清理；本机控制中心新增统计、筛选、分页和 CSV 导出，管理页面不通过公网开放。
- 公共站新增 `/privacy` 隐私说明页，公共查询表单同步展示记录范围和保存期限，并加入 Sitemap。
- 实时风险协议升级到 `2026-07-21-v3`，明确区分“不保存原始 IP 的风险确认记录”和“保存 IP 的功能使用审计”；旧版本确认需重新完成。

## 2026-07-20

- 实时对局新增强制风险知情确认：显著说明流量处理、代理证书、账号限制和临时或永久封禁风险。
- 新增独立 `DISCLAIMER.md`，明确有限测试未发现封号不等于零风险、用户合规义务、第三方处罚边界和依法不得免责的责任。
- 风险确认改为三项独立复选框；服务端保存协议版本、正文摘要、确认时间和去标识化环境摘要，确认有效期 180 天。
- 未确认时，连接命令、账号选择、房间、当前对局及公开轮询接口均不可使用，不能通过直接访问 URL 绕过。
- 新增“一键撤回并永久锁定”：原确认记录保留并追加撤回时间，当前浏览器此后不再提供重新确认或实时功能入口。
- 重新设计 GitHub 项目首页，按产品效果、场景价值、快速开始、真实截图、工作原理和隐私边界组织内容。
- 新增独立使用教程 `docs/USAGE.md`，覆盖昵称查询、实时连接、账号选择、房间近 3 场、当前对局 10 人、账号切换、网络恢复和常见故障。
- 公共查询首页新增醒目的实时对局入口，并上线独立的 `/live` 功能页。
- 实时页公开展示连接、验证与恢复网络步骤，用户无需管理口令即可使用公共只读功能。
- 网吧多人共享公网 IP 时，不再自动加载该 IP 下最新对局；页面改为列出同网络有效账号，由用户按游戏昵称选择。
- 账号选择使用与当前网络绑定的临时标识和浏览器 Cookie 保存，不向页面暴露内部 session、subject、PartyID、原始事件 ID 或 token 来源。
- 房间页与当前对局页只读取用户主动选择的账号；未选择时不会自动查询。
- 当前对局左侧英雄栏新增疑似组队彩色连线：每个疑似小队使用独立颜色的头像外圈、短分支、纵向连线和“组队 N”标签。
- 疑似组队只采用同阵营玩家近期作为友方共同参赛的记录，并将相互关联的玩家合并为同一小队。
- 公共实时结果页保持 `noindex`，公共查询页与实时功能介绍页加入站点地图，兼顾搜索收录和动态对局隐私。
- 修订项目记忆、API、架构、运维与交接文档，并完成桌面端、窄屏和真实 10 人当前对局验证。

- README 与截图文档换成脱敏后的真实运行页面。
