# 使用教程

这份教程面向第一次使用 WGame VAL Query 的玩家，覆盖昵称查询、实时房间、当前对局、账号切换和常见问题。

## 入口

| 功能 | 地址 | 是否需要连接游戏 |
| --- | --- | --- |
| 昵称#ID 战绩查询 | [https://val.skzspro.cn/public](https://val.skzspro.cn/public) | 不需要 |
| 房间与当前对局 | [https://val.skzspro.cn/live](https://val.skzspro.cn/live) | 需要按页面步骤连接 |

## 一、查询某个玩家的近期战绩

### 第 1 步：打开公共查询页

访问：

[https://val.skzspro.cn/public](https://val.skzspro.cn/public)

### 第 2 步：输入完整昵称

输入格式：

```text
游戏昵称#数字ID
```

示例：

```text
GGbone#55989
```

昵称和 `#` 后面的数字 ID 都需要完整填写。

### 第 3 步：查看结果

查询结果通常包含：

- 最近 10 场胜负
- 每场使用英雄
- 比分和 K/D/A
- ACS
- 地图
- 段位与 RR 变化
- 对局时长和高光
- 数据接口返回的公开省级 IP 属地

## 二、连接实时房间和当前对局

### 第 1 步：打开实时入口

访问：

[https://val.skzspro.cn/live](https://val.skzspro.cn/live)

### 第 2 步：按页面完成连接

实时页会依次展示：

1. 安装连接所需证书。
2. 配置网络连接。
3. 验证连接状态。
4. 使用结束后的恢复方式。

请以实时页面当时显示的命令为准。页面提供复制按钮，不需要手动重写命令。

### 第 3 步：启动或保持游戏在线

游戏产生新的有效会话后，实时页会显示当前网络下可选择的游戏账号。

如果暂时没有看到账号：

- 确认游戏仍在运行。
- 确认已经完成实时页的连接步骤。
- 等待游戏产生新的网络请求后刷新页面。
- 确认浏览器和游戏使用的是同一个网吧出口网络。

### 第 4 步：按昵称选择自己的账号

网吧内多台电脑可能共享一个公网 IP，因此不能只靠 IP 判断你是哪一名玩家。

页面会列出同网络下仍有效的游戏账号。请：

1. 查看游戏昵称。
2. 选择属于自己的账号。
3. 再进入房间或当前对局页面。

选择结果会在当前浏览器中临时保存。页面不会自动选择同 IP 下更新时间最新的账号。

### 第 5 步：根据游戏阶段进入功能

| 当前阶段 | 推荐入口 | 可以看到什么 |
| --- | --- | --- |
| 正在找人组队 | 房间近 3 场 | 当前房间其他成员近期状态 |
| 已开始匹配 | 房间近 3 场 | 房间成员和匹配补入队友 |
| 进入选人或对局 | 当前对局 10 人 | 双方玩家、英雄和近 10 场 |

## 三、房间近 3 场怎么使用

进入房间页后，系统会：

- 排除当前账号。
- 加载其他成员近 3 场。
- 标注“房间成员”或“匹配队友”。
- 玩家加入或退出时自动同步。
- 匹配成功后补齐其他我方玩家。
- 全部加载完成后生成群聊文本摘要。

如果房间成员发生变化，页面会自动重建当前列表，不需要反复手动打开新页面。

## 四、当前对局 10 人怎么看

### 左侧英雄栏

- 上下分组表示我方和敌方。
- 蓝色 / 红色内圈用于识别阵营。
- 点击某名玩家可直接跳转到他的战绩。
- “组队 N”及同色外圈、连接线表示疑似组队关系。

### 玩家战绩区

每名玩家的数据会独立加载。已经完成的玩家会先显示，不需要等待全部 10 人一起完成。

常见字段包括：

- 最近胜负
- 当前英雄
- 段位
- K/D/A
- ACS
- 地图
- RR 变化
- 高光信息

### 疑似组队

系统比较同阵营玩家近 10 场中的共同对局：

1. 找到双方都出现过的对局。
2. 判断当时是否作为友方共同参赛。
3. 将相互有关联的玩家合并成疑似小队。
4. 用不同颜色在左侧英雄栏连接。

这只是辅助推断，不是官方 PartyID。偶遇、连续排到同一批玩家等情况也可能产生共同对局。

## 五、切换游戏账号

已经选择账号后，可以在实时页面点击“切换游戏账号”。

适用情况：

- 同一台电脑更换了游戏账号。
- 之前误选了网吧内其他玩家。
- 当前账号退出后重新登录。
- 页面仍保留上一次临时选择。

切换后，房间页和当前对局页都会使用新的明确选择。

## 六、使用结束后恢复网络

离开网吧、停止使用实时功能或不再需要连接时，请回到实时页，执行页面提供的恢复步骤。

恢复后可以再次打开实时页检查状态，确认浏览器和系统网络已经回到正常连接方式。

## 七、常见问题

### 页面一直显示“等待游戏连接”

可能原因：

- 游戏还没有产生新的有效会话。
- 连接步骤没有完成。
- 会话已经过期。
- 浏览器与游戏使用了不同的出口网络。

建议重新核对实时页的连接和验证步骤，再启动游戏或进入一个房间。

### 能看到多个账号

这是网吧共享公网 IP 时的正常现象。

不要按更新时间猜测，直接按游戏昵称选择自己的账号。

### 房间页没有其他玩家

确认自己已经进入多人组队房间。单人大厅、成员尚未加入或游戏状态切换期间，列表可能暂时为空。

### 当前对局页提示未检测到对局

当前可能仍在大厅、组队或匹配阶段。进入选人或正式对局后再刷新。

### 没有出现疑似组队彩色连线

彩色连线需要：

- 当前玩家近期战绩加载完成。
- 同阵营玩家存在可确认的友方共同对局。

没有共同记录时不会强行生成关系。

### 查询结果和游戏内当前段位略有不同

战绩中的段位可能是该局记录时的段位，不一定等于此刻最新段位。

## 八、隐私提醒

- 页面不展示玩家真实 IP。
- 同网络 IP 只用于筛选候选账号。
- 未选择账号时不会自动加载实时对局。
- 公共页面不返回内部 session、subject、PartyID 或 token 来源。
- 不要在 GitHub Issue 中提交 Cookie、token、抓包文件或登录二维码。
